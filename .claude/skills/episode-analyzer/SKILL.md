# episode-analyzer: わせだや エピソード分析スキル

ユーザーが日常で仕入れた面白いエピソードを分析し、「時代別あるある」台本の素材に変換する。neta-developerの「テーマ → ネタ5案」に対し、このスキルは「具体エピソード → 1本の台本素材」を作る。

## 核となる思想

- エピソードの **「なぜ面白いのか」** を言語化する。笑いの構造を分解しないと台本に活かせない
- エピソードを **そのまま使うのではなく「あるある」に昇華** する。個人の体験 → 万人に刺さる共感へ
- エピソードが最も輝く年代を見極め、他年代との **対比構造** を設計する

## ターゲット理解

neta-developerと同じ:
- 20代後半〜40代の激務サラリーマン（男性中心）
- 「年収」「昇進」「転職」「住宅」「結婚」「子育て」が常に頭にある
- 通勤電車・昼休みにスマホで見る

## 感情トリガー（エピソードがどれに当たるか判定する）

1. **共感**「分かるわー」
2. **世代間ギャップ**「上の世代マジでこう」
3. **自虐**「俺らの世代きつくね？」
4. **ノスタルジー**「あの頃は」

## 戦略ドキュメント連携

Google Driveの「わせだや/戦略/strategy.md」が存在する場合、読み込んで参照する:
- エピソードから抽出したテーマが「避けるべき方向性」に該当しないか確認
- 該当する場合はテーマの切り口を調整する（エピソード自体は活かしつつ）
- strategy.md が存在しない場合でも正常に動作する

## 処理ステップ

### ステップ1: エピソード分解

- **何が起きたか**: 事実を1-2行で要約
- **なぜ面白いか**: 笑いの構造を分析（ギャップ、皮肉、あるある、自虐、予想外の展開 など）
- **誰に刺さるか**: ターゲット（激務サラリーマン）との接点

### ステップ2: テーマ抽出

エピソードから時代横断できる抽象テーマを抽出する。

- タイトルは「時代別、〇〇〇〇」の形式
- エピソードの面白さの核を保ちつつ、各年代で比較可能なテーマにする
- 横軸は年代別（1990/2000/2010/2025）を基本とする

**例**:
- エピソード「上司に転職って言ったら翌日優しくなった」→ テーマ「時代別、部下の退職を引き止める方法」
- エピソード「年収1000万なのに1LDK」→ テーマ「時代別、年収1000万円の生活レベル」

### ステップ3: 年代アサイン

エピソードが最もハマる年代を判定する。

**判定基準**:
- エピソードの状況が最もリアルに成立する年代はどこか
- 経済状況・社会通念・価値観がエピソードの前提と合致するか
- そのエピソードの口調・空気感がどの年代に近いか

**判定結果の年代別傾向**:
- 1990年に配置: バブル的な豪快さ・金銭感覚が絡むエピソード
- 2000年に配置: 不景気・我慢・卑屈さが絡むエピソード
- 2010年に配置: そこそこ安定・スマートさが絡むエピソード
- 2025年に配置: 物価高・転職・副業・Z世代が絡むエピソード（迷ったらここ）

### ステップ4: 台本への申し送り

下流のjidai-researcherとscript-writerに渡す情報を整理する。

## 出力フォーマット（必ずこの形式で出す）

```
## エピソード分析

### 元エピソード
（ユーザーの原文をそのまま記載）

### 分析
- おもしろポイント: （なぜ面白い/共感できるのか。笑いの構造を1-2行で）
- 感情トリガー: 共感 / 世代間ギャップ / 自虐 / ノスタルジー（該当するもの）
- ターゲット刺さり度: ★★★★☆（理由1行）

### テーマ抽出
- タイトル: 時代別、〇〇〇〇
- 抽象テーマ: （エピソードから抽出した時代横断テーマ1行）
- 横軸: 年代別（1990/2000/2010/2025）

### 年代アサイン
- エピソード配置年代: 【20XX年】
- 配置理由: （なぜこの年代が最適か1-2行）

### 台本への申し送り
- エピソードの核: （台本に必ず残すべき面白さの要素）
- アダプテーション方針: （エピソードを配置年代の口調に変換する方向性）
- 他年代の方向性:
  - 【1990年】: （この年代での同テーマの切り口1行）
  - 【2000年】: （同上）
  - 【2010年】: （同上）
  - 【2025年】: （同上）
  ※エピソード配置年代は「エピソードを活用」と記載
- 推奨オチパターン: A（共感リアル）/ B（ドライ）/ C（逆転）から選択（理由）

### 戦略整合
（strategy.mdのどの傾向に合致するか。strategy.mdがない場合は省略）

バズ予測: ★★★★☆
```

## チェック（出力前に必ず確認）

- [ ] エピソードの面白さの核を正しく抽出できているか？
- [ ] 抽出テーマは4年代で比較可能か？（エピソードに引っ張られすぎて狭くなっていないか）
- [ ] 年代アサインに根拠があるか？
- [ ] 他年代の方向性が、エピソード配置年代との対比になっているか？
- [ ] ターゲット（激務サラリーマン）が電車で見て興味を持つタイトルか？

## テーマ適合度が低い場合

エピソードが「時代別あるある」の形式に合わない場合（例: 時代横断が難しい超個人的な話）:
- テーマの切り口を広げて対応を試みる（「上司が〜した」→「時代別、上司の〇〇」に拡張）
- それでも難しい場合は、ユーザーに「このエピソードは時代別の構造に合わせづらいため、切り口を変えます」と伝えた上で最善のテーマを提案する
