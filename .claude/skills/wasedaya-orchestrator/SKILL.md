# wasedaya-orchestrator: わせだや 台本全自動生成 統括スキル

テーマを一言投げるだけで、ネタ開発→リサーチ→台本執筆→レビュー→Google Drive保存を一気通貫で実行する。

**重要**: このスキルにはドメインロジックを書かない。各スキルのSKILL.mdの指示をそのまま実行する。

## 処理フロー

### ステップ1: ネタ開発（neta-developer）

`/neta-developer` の指示に従い、ネタ5案を生成する。

- ユーザーからテーマ指定がある場合: そのテーマで5案
- テーマ指定がない場合: 自由にネタ5案
- **strategy.md連携**: Google Driveの「わせだや/戦略/strategy.md」が存在する場合は読み込んで参照する。存在しない場合はスキップして続行。

### ステップ2: ベスト案の自動選定

5案の中からバズ予測★が最高の案を自動選定する。
- 同点の場合は「予測と裏切り」の切れ味で判断する
- 選定理由を1行で記録する

### ステップ3: 時代リサーチ（jidai-researcher）

`/jidai-researcher` の指示に従い、選定したネタのリサーチシートを作成する。

### ステップ4: 台本執筆（script-writer）

`/script-writer` の指示に従い、リサーチシートを元に台本を執筆する。

### ステップ5: レビュー＆自動修正（script-reviewer）

`/script-reviewer` の指示に従い、台本をレビューする。

**自動修正ループ**:
1. レビュー結果に🟡🔴の軸がある場合 → script-reviewerの修正版を採用
2. 修正版を再度レビュー（2回目）
3. まだ🟡🔴がある場合 → もう一度修正＆レビュー（最大2回まで）
4. 全軸🟢、または2回修正後 → その時点の台本を最終版とする

### ステップ6: Google Driveに保存

Google Drive MCPを使用して最終台本を保存する。

**保存先**: 「わせだや/台本/レビュー待ち/」フォルダ

**ファイル名**: `YYYYMMDD_タイトル`（例: `20260301_時代別、転職活動の違い`）

**ファイル内容の構造**:
```
タイトル: 時代別、〇〇〇〇
生成日: YYYY-MM-DD
バズ予測: ★★★★☆
テーマタグ: （テーマを2〜3個のタグで）
戦略整合: （strategy.mdのどの方向性に合致するか。strategy.mdがない場合は「なし」）
レビュー結果: テンポ🟢 口調🟢 裏切り🟢 あるある🟢 ドーピング🟢

===

（台本本文）

===

【生成メモ】
選定理由: ...
修正履歴: （レビューで修正した内容のサマリー。修正がなければ「初回で全軸🟢」）
```

## 最終出力フォーマット

デフォルトでは最終結果だけ返す:

```
## 生成レポート
選定ネタ: （タイトル）
選定理由: （1行）
戦略との整合: （strategy.mdのどの方向性に合致するか1行。strategy.mdがない場合は省略）
レビュー結果: テンポ🟢 口調🟢 裏切り🟢 あるある🟢 ドーピング🟢

---
（最終台本の全文）
---

Google Driveに保存しました: [ファイル名]（リンク）
```

**「詳しく見せて」「途中経過も」と言われたら**: 各ステップ（ネタ5案、選定理由、リサーチシート、初稿、レビュー結果、修正履歴）をすべて表示する。

## エラーハンドリング

- **Google Drive MCPが接続されていない場合**: 台本は生成するが保存はスキップ。「Google Drive MCPが接続されていないため保存をスキップしました。台本はこのチャット内に出力しています。」と表示。
- **strategy.mdが見つからない場合**: strategy.md連携をスキップして通常どおりネタ開発を行う。
- **レビューで全軸🔴の場合**: ネタ選定からやり直すことを提案する。
